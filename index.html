K Nearest Neighbours

Pseudocode

$\text{ X $\leftarrow$ training data; Y $\leftarrow$ class labels; x $\leftarrow$ unknown sample; k $\leftarrow$ considered neighbours}$
$\text{for i = 1 to m do}$
$\text{ $\quad$ compute distance d(Xi, x)}$
$\text{end for}$
$\text{Compute set I containing indices for the k smallest distances d(Xi, x)}$
$\text{return majority label for \{Yi where i $\in$ I \} }$

Description

K Nearest Neighbour is an instance-based algorithm: it doesn't build a model to perform classification but instead bases its predictions on the training data stored in its memory.
The method takes in input a training sample with attribute values and labels and one or more records to be classified.
Since the label is assigned according to the most similar records' classes in the training set, a value k, indicating how many neighbours consider, is required. A criterion to measure distance is also needed.
During the execution, the algorithm computes the distance between every record to classify and every row in the training set, and keeps the k nearest ones.
The label is assigned according to the most frequent between the k nearest neighbours. Sometimes a weight is added in order to give less importance to the most marginal records.
KNN is considered a lazy learner because instead of generalising data it compares each times the record to be classified with every data in the training set. This can be expensive because a lot of distance calculations are needed but in the same time no effort is needed to build a model.
A critical issue of this algorithm is that a wrong value of k could lead to bad classification: in fact, a too low k could make the classification susceptible to noise, while a too big k could hinder it to find interesting patterns.
